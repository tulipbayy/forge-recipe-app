Developer Setup Guide
---------------------
Installation:

cd frontend
npm install 
npm run dev 

cd backend
npm install 
node index.js

Required Environment Variables:

Frontend:
Create a .env file in frontend/, format it like this:

VITE_FIREBASE_API_KEY="..."
VITE_FIREBASE_AUTH_DOMAIN="..."
VITE_FIREBASE_PROJECT_ID="..."
VITE_FIREBASE_STORAGE_BUCKET="..."
VITE_FIREBASE_MESSAGING_SENDER_ID="..."
VITE_FIREBASE_APP_ID="..."

*These are located in the Firebase console

Backend:
Create a .env file in backend/, format it like this (I set it to Edamam for now, we can change it if we use a different API):

EDAMAM_APP_ID="..."
EDAMAM_APP_KEY="..."
EDAMAM_ACCOUNT_USER="..."

*I invited you as users for the API, let me know if you didn't get the invite.

Download serviceAccountKey.json from the Firebase console and put it in backend/. MAKE SURE TO ADD IT TO .gitignore!!!!!!

Other notes:
------------
We are using Tailwind v4, it is already configured directly inside frontend/vite.config.js and src/index.css

The backend uses import / export syntax, not require()

If you need to read/write to Firestore in backend routes, do not re-initialize Firebase. Just import it like:

import { db } from '../firebaseAdmin.js';
